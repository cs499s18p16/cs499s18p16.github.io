Design Considerations

Information to relate to the reader for issues relating to design considerations or the implementation:

  • Issues/problems during implementation
  • Attempted to use Amazon Web Services to host the web application
  • No sufficient training in the web hosting service led to complications in launching the project
  • Lost out on time implementing actual code due to time spent configuring AWS
  • Unfamiliarity with MySQL and PHP languages led to some delays in progressing through the project
  • Changes to design necessary during implementation
  • Discussed and collectively agreed as a group to switch from hosting the web application on AWS to a LAMP stack using the multilab servers provided to us through UK
  • This change allows for a simpler installation and more customizability for the customer
  • Reorganization of the GitHub repository due to redundant code/files and difficulty navigating through the project

Future Enhancements

What the next group can look to build upon.

  • Implementing more functionality into the web application on top of the availability tool
  • Ability for Scrum Master and Product Owner to track capacity and velocity per sprint in respect to their teams
  • Allow Product Owner to attach specific details in regards to a project
  • Set a start and end date for each project assigned
  • Give a description of the project so developers know exactly what they are working on
  • Allow all users to have a customizable profile page that includes details about them and the projects they have worked on
  • Improve security of the web application
  • Allow for new users added, regardless of role, to reset their passwords once added to the system
